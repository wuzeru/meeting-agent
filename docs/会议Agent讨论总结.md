# 会议 Agent 产品讨论纪要

本文档整理自关于「可追问、可探讨的会议 Agent」的讨论，供后续产品与方案参考。

## 一、要做什么

目标不是只做转写和纪要，而是像真人同桌一样参与讨论：适时追问、澄清矛盾、归纳共识与分歧，帮助团队发现真正价值，并把结论落成可执行的下一步。纪要应是副产品，而非唯一产出。

## 二、价值是什么

把会里的话说清楚、记下来，减少漏听漏记；更强一层是当场把零散讨论拧成结论、待办与责任边界，缩短从「聊完了」到「能干活」的时间，提升对齐效率与决策质量。

## 三、市面大厂在做什么（产品名与侧重点）

内部研发代号大多不公开，对外多为正式产品名，功能以辅助与记录为主，「主动 facilitator」仍偏克制。

- 微软：Microsoft 365 Copilot；Teams 中的 Copilot（会后纪要、智能回顾、按记录提问等）；Facilitator 会议智能体（议程与进度、协作笔记、任务进 Planner、按讨论起草 Word/Loop、会议聊天中可 @ 问答等）。
- 谷歌：Gemini in Google Meet；会议中的「问问 Gemini」；「帮我记笔记」（落地 Google 文档、待办、挂日历；部分场景有迟到补看的当场摘要）。
- Zoom：AI Companion（及 Custom AI Companion 等），覆盖会议与协作场景中的摘要与办公辅助等。
- 思科 Webex：Webex AI Assistant（摘要、待办与会中会后辅助等）。

整体上，成熟能力多集中在实时/会后摘要、问答与任务提炼；像真人一样深度引导、持续探讨的产品化程度仍有限，且受合规与打断体验约束。

## 四、有没有必要单独做

有必要从「追问与探讨」上差异化，避免只跟大厂拼纪要红线；同时必须锁定愿意为解决对齐成本、决策质量与留痕合规而付费的场景，并把打断节奏、误判风险与隐私合规做稳。

## 五、愿付费的典型场景

高层拍板会、大额销售与投委会、咨询共创与复盘、以及合规要求强留痕、专家时间成本高的会议。共同点是：讨论飘着成本高、最怕说不清责任人、时间节点与验收标准。

举例：企业软件采购复盘类会议中，销售与客户技术负责人信息各半，Agent 可协助追问验收标准、决策人、预算与上线时间等，把口头共识钉进纪要与待办；对这类会，时间延误与返工成本往往显著，付费意愿更强。

## 六、资深人类主持人 vs Agent

资深主持人仍具优势：气场、信任与现场判断。但人贵、难随到、也会漏听或带偏见。Agent 更适合作为放大器：全场一致记录、结构化追问清单、会后跟踪与版本化结论。较理想形态是人控节奏与关系，Agent 打底稿、查漏与执行清单。

## 七、客户需求探询类会议

人在前建立信任、察言观色；Agent 在后按框架补问缺口、记原话与矛盾点，散会可快速产出需求清单与待确认项。它不能替代关系与温度，也需在客户明显反感时避免强硬打断，可配合「侧边栏提示 + 人工点发送」等形态。

## 八、与「智能客服」是否一类

不是一类。智能客服多为用户主动发起、按知识库与流程的一对多应答与工单；会议 Agent 面向多人实时流式对话，要理解上下文与话轮、择机介入，目标是 facilitation 与对齐，而不是排队解答标准问题。

## 九、择机追问的实现思路（概要）

核心是维护随会议推进更新的「开放问题 / 缺口」清单，实时标记含糊表述、矛盾、缺关键决策人、久拖不决等；对每个缺口按重要性、敏感性、当前话轮是否适合打断进行打分，再决定是否发声。工程上需重视话轮检测、静音与打断礼貌策略；早期可采用先推侧边栏建议、由主持人确认后再宣读或发送，降低冒犯与误判，成熟后再逐步自动化。

## 十、一句话归纳

差异化在「会中引导与对齐」而不只纪要；落地要场景聚焦、人机分工清晰、合规与体验可控，技术上以结构化缺口检测 + 打断策略和产品形态渐进释放为主。
